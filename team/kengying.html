Enhancement Added: Group

External behavior

Start of Extract [from: User Guide]

Adding a person: add

Adds a person to the address book
Format: add n/NAME g/GROUP [p/PHONE_NUMBER] [e/EMAIL] [a/ADDRESS] [t/TAG]…​ [c/CUSTOMFIEILD_KEY:CUSTOMFIELD_VALUE]…​

A person can have any number of tags or custom fields (including 0)
A person can be saved with just name and group

Examples:

  • add n/John Doe p/98765432 e/johnd@example.com a/John street, block 123, #01-01 g/Health c/School:NUS c/Company:Google

  • add n/Betsy Crowe t/friend e/betsycrowe@example.com a/Newgate Prison p/1234567 g/Life t/criminal

  • add n/Henry Harry g/Savings

  • add n/Tim Tom p/12356923 a/Timmy street g/Holiday

End of Extract

Justification

User will be able to add group to their different clients. The difference between tag and group is that, a user must have a group when being added while tag is optional.

Implementation

Start of Extract [from: Developer Guide]

Group

The add command allows user to add group to a person. Every person will need to have a group.

Design Considerations

Aspect: Implementation of Group
Alternative 1 (current choice): Use the current add command for adding of groups.
Pros: Easier to be done.
Cons: Will not be able to create an empty group.
Alternative 2: Create another command for managing groups.
Pros: More distinct differentiation of group and tag.
Cons: Time consuming.

End of Extract

Enhancement Added: List

External behavior

Start of Extract [from: User Guide]

[Since v1.3] Listing all persons : list

list all, shows all person in humanDEX
list tags, shows all tags in humanDEX
list groups, shows all groups in humanDEX
Format: list [SEARCH_TERM]

Examples:

  • list all

  • list tags

  • list groups

End of Extract

Justification

As the amount of contacts increase, there will be too many tags and groups to keep track. List tags/groups, will be able to show the available tags/groups that are currently being in used.
